graphiCAL

My newest, self-made calculator. I have used graphics displays with OLED technology in other works. To my surprise, the control of these displays is very similar to the control of graphics LCDs. (Cacula and MINICAL calculators have 132*32 pixel graphics LCDs.) There is a slight discrepancy in ininformaizing. This gave me the idea to provide my new machine called graphiCAL with this display. The 128*64 pixel resolution allows you to display 16 “big” characters in 4 rows, or 21 “small” characters in every line.

All calculators I produce use so-called Reverse Polish Marking when performing operations. RPN (Reverse Polish Notation) This name was given to the procedure in honor of a Polish mathematician, Jan ?ukasiewicz (Ján Vukasevics). ?ukasiewicz developed a formal logical system that allowed mathematical expressions to be entered without parentheses by placing the operators in front of the prefix notation (postfix notation) in front of the operands (postfix notation). There are a lot of literature on RPN on the Net. One of the many: https://www.hpmuseum.org/rpn.htm .

As you can see in the image on the left, all four Stack registers can be displayed above the usual X-register. There is also room for different signs.

Some features of graphiCAL:

  • PIC18F67K22 microcontroller-built graphic, programmable calculator,
  • Display: 128*64 pixels OLED (color: orange, sky-blue or white),
  • input 43 keys in the "traditional" hp layout,
  • intake: incremental transmitter (increased, gradual) edge button (thumb-wheel),
  • calculator housing: 3D printed,
  • Operations: RPN (Reverse Polish Notation),
  • Operations: ASMD*, transcendent functions, n!, x?2, yx, ?, etc. **
  • number storage: 29 registers (ASMD operation) + LASTx + Indirect + Flags**,
  • programming: with mnemonic (easy to remember) alphanumeric codes,
  • program editing: line deletion, row-payment, line-up, overwriting, etc.,
  • unconditional and conditional jump, jumping to absolute and labeling,
  • Number of subroutines: 16,
  • program length: 256 steps,
  • Number of programme steps: 8
  • special functions: astronomical, mathematical, chemical and physical constants, US/Metric conversion,
  • "external" program repository (48L512N RAM/EEPROM -> EERAM) 64 programs,
  • built-in RTC: heat-compensated RV-3049-C2,
  • sound: 35 mm Piezo hum,
  • Connection: USB3-C, for battery charging and updating the program,
  • Program Update: Microchip Serial Bootloader with AN1310 v.1.05r (WIN XP.... WIN 10),

*: ASMD: Addition Subtraction Multiplication Division -> Addition Withdrawal

**: Detailed description in separate, downloadable documents.

Hardware Documentation

The graphiCAL is designed on a single double-sided, bore-galvanized printed wiring disc (PCB). The “Gerber” files required for production are contained in . The drawings in principle are included in the graphiCAL_sch file. 5 pieces of A4 size png file. On page 5, only the PIC18F67K22 microcontroller version of the TQFP leg design is shown. Since the TQFP and QFN enclosures are leg-compatible with this type, any one can be used in the circuit. (Whatever you can get.)

Plate thickness: 1,2 mm

Marking: (Gerber files are made in Gerber_RS274x format.)

  • graphical_top.gbr : spare parts side (TOP) wiring
  • graphical_bot.gbr : soldering side (BOTtom) wire
  • graphical_tmk.gbr : spare parts anti-sold mask (Top MasK)
  • graphical_bmk.gbr : soldering side mask (Bottom MasK)
  • graphical_tsk.gbr : dentical position drawing (Top SilK)
  • graphical_cut.gbr : erosion instruction (There are 4 short milling inside the panel due to the fixing of the USB-C connector.)
  • graphical.drl : drilling instruction in Excellon format

The case of is made with 3D printing. STL files (in a compact form): kovax.co.hu/...3/05/graphiCAL_STL_files.zip

These parts were designed so that the application of a separate support material printing technology was the consideration.

The graphiCAl part list can be downloaded in graphiCAL21_BOM pdf format. This is the list of the 2.1 HW version. Versions 2.0 and 2.1 differ only in the type of positioning button. In the earlier version, it was possible to position the pointer (cursor) by waving a so-called edge button. Unfortunately, I can no longer purchase this Panasonic-made EVQ-WKA type, I can’t get this part anymore. (Perhaps it is available on AliExpress.) The English name is thumb wheel. Instead, in version 2.1, a new positioner was replaced. This is a switch made by Multicomp, MCPL3-AC-V type, type, up/down. The English name of this is: law dial, right wheel... Of course, there is also a push button function.

There is even an important part of the graphiCAL that can be made at home. And this is the keyboard label. I use the following production process: I print the graphiCAL_keyboard file in pdf format in good quality on a white paper with self-adhesive film. After removing the cut-out film, I glue it to a cooling magnet sheet with a thickness of about 1 mm. (Can be purchased in A4 size.) The magnetic film must be glued to the side of the one that does not stick to the refrigerator. I still coated this “sandwich” with a water-cleaned self-adhesive film for the protection of the surface. I puncture it at the keys with a 4mm leather punch shop, which is a small circle in the drawing. After that, it is only necessary to cut the front panel to size with a sharp knife.
